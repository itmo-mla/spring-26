# Описание датасета

В качестве данных использован набор **[MovieLens ml-latest-small](https://grouplens.org/datasets/movielens/)** — классический benchmark для коллаборативной фильтрации с явными оценками и текстовыми метаданными о фильмах.

Источник: [GroupLens MovieLens](https://grouplens.org/datasets/movielens/latest/). Файлы лежат в `source/ml-latest-small/`.

| Характеристика | Значение |
| -------------- | -------- |
| Пользователей | 610 |
| Фильмов (в `ratings.csv`) | 9 724 |
| Оценок | 100 836 |
| Диапазон рейтингов | 0.5–5.0 |
| Минимум оценок на пользователя | ≥ 20 |

Для обучения используется `ratings.csv` (колонки `userId`, `movieId`, `rating`). Текстовая информация о фильмах доступна в `movies.csv` (название, жанры); в экспериментах по RMSE/NDCG используются только числовые взаимодействия «пользователь–фильм».

Предобработка (`utils.py`):

- дедупликация пар `(userId, movieId)` со средним рейтингом;
- матрица взаимодействий **user × item** (`csr_matrix`);
- разбиение 80/20 по парам (`random_state=42`): train — 80 668 оценок, test — 20 168.

# Реализация SLIM

**SLIM** (Sparse Linear Methods) — item-based модель: для каждого фильма $i$ подбирается разреженный вектор весов $w_i$ по остальным фильмам, так что столбец оценок $r_i$ аппроксимируется линейной комбинацией других столбцов.

$$
\min_{w_i} \;\frac{1}{2}\|r_i - R w_i\|_2^2 + \frac{\beta}{2}\|w_i\|_2^2 + \lambda \|w_i\|_1
\quad \text{при} \quad w_i \ge 0,\; w_{ii} = 0
$$

Предсказание: $\hat{R} = R W$, где $W \in \mathbb{R}^{n_{\text{items}} \times n_{\text{items}}}$.

Собственная реализация: `source/slim.py` — Coordinate Descent через `sklearn.linear_model.ElasticNet` по столбцам матрицы $R$.

Эталон: [KarypisLab/SLIM](https://github.com/KarypisLab/SLIM) (`source/SLIM/`), обёртка `reference_slim.py`. Сборка: `bash source/build_reference_slim.sh`.

Гиперпараметры:

```
SLIM(l1_reg=1e-1, l2_reg=1e-1, max_iter=100)
ReferenceSLIM(l1r=1e-1, l2r=1e-1, nthreads=2, max_iter=100)  # algo='cd'
```

# Латентная модель: ALS

В качестве латентной семантической модели реализован **ALS** (Alternating Least Squares) — матричная факторизация с явными рейтингами и смещениями:

$$
\hat{r}_{ui} = \mu + b_u + b_i + p_u^\top q_i
$$

Чередуются шаги: обновление факторов пользователей, факторов фильмов, смещений $b_u$, $b_i$.

Собственная реализация: `source/als.py`.

Эталон для сравнения: **implicit ALS** (`implicit` library), обёртка `reference_als.py` — оптимизирует implicit-feedback задачу с confidence $c_{ui} = 1 + \alpha r_{ui}$, предсказание для RMSE: $P Q^\top$.

Гиперпараметры:

```
ALS(n_factors=50, reg=1e-1, n_iter=100)
ImplicitALS(factors=50, regularization=1e-1, iterations=100, alpha=40)
```

# Метрики и протокол оценки

- **RMSE** на hold-out парах из test (`utils.rmse_on_test`): модель обучается только на `R_train`, предсказание не использует test-оценки при обучении.
- **NDCG@10** (`utils.ndcg_at_k`, задача со *): для каждого пользователя с оценками в test строится ранжирование по скору модели; уже просмотренные в train фильмы исключаются; релевантность = значение рейтинга в test.

Код экспериментов: `source/data_preprocess.ipynb`.

# Результаты экспериментов

## RMSE

| Модель | Train RMSE | Test RMSE |
| ------ | ---------- | --------- |
| SLIM (своя) | 2.719 | **3.019** |
| SLIM (KarypisLab) | 0.924 | 3.331 |
| ALS (своя) | **0.015** | **1.266** |
| ALS (`implicit`) | 2.726 | 3.141 |

Своя ALS минимизирует квадратичную ошибку по рейтингам напрямую, поэтому RMSE на MovieLens существенно ниже.

SLIM и эталонный SLIM по test RMSE сопоставимы (≈3.0–3.3): модель слабее восстанавливает абсолютную шкалу 0.5–5, но улавливает item–item структуру.

`implicit` ALS оптимизирует другую цель (взвешенный implicit feedback); dot-product не калиброван под звёзды — высокий test RMSE ожидаем.

## NDCG@10

| Модель | NDCG@10 |
| ------ | ------- |
| SLIM (своя) | 0.331 |
| SLIM (KarypisLab) | 0.248 |
| ALS (своя) | 0.055 |
| ALS (`implicit`) | 0.164 |

Для ранжирования по скрытым в test фильмам SLIM показывает более высокий NDCG@10, чем факторизация ALS при тех же гиперпараметрах. Низкий NDCG у явного ALS связан с тем, что модель настроена на точное восстановление оценок, а не на top-K порядок; при подборе гиперпараметров под ranking-метрику картина может измениться.

# Сравнение с эталонными реализациями

### SLIM

| Аспект | Своя реализация | KarypisLab SLIM |
| ------ | --------------- | --------------- |
| Test RMSE | 3.019 | 3.331 |
| NDCG@10 | **0.331** | 0.248 |
| Время обучения (9724 фильма) | ~59 с | ~62 с |
| Оптимизатор | sklearn ElasticNet | Coordinate Descent (C++) |

Направления совпадают: обе модели дают близкий test RMSE. Различия объясняются масштабированием регуляризации в sklearn ($1/(2n)$ в функции потерь) и разной численной реализацией CD.

### ALS

| Аспект | custom ALS | `implicit` ALS |
| ------ | -------- | -------------- |
| Test RMSE | **1.266** | 3.141 |
| NDCG@10 | 0.055 | **0.164** |
| Задача | явные рейтинги + biases | implicit confidence |

Сравнение по RMSE для `implicit` некорректно как «чистое» сопоставление алгоритмов: библиотека не обучается минимизировать MSE по звёздам. Для отчёта эталон `implicit` приведён как промышленная реализация ALS; основная латентная модель с явным RMSE — собственная `als.py`.


# Выводы

- Реализованы SLIM (item-based sparse linear) и ALS (latent factorization с biases) на MovieLens; пайплайн оценки — hold-out RMSE и NDCG@10.
- По **test RMSE** лучший результат у явной ALS (1.27); SLIM даёт ~3.02 и близок к эталону KarypisLab (3.33).
- По **NDCG@10** среди протестированных моделей лидирует своя SLIM (0.33); эталонный SLIM и `implicit` ALS полезнее интерпретировать в рамках своих постановок (ranking / implicit).
- Эталонные реализации подключены и воспроизводят порядок величин на том же split; расхождение `implicit` по RMSE отражает разную целевую функцию, а не ошибку интеграции.
- Для production-рекомендаций на явных рейтингах предпочтительна факторизация ALS; SLIM уместен, когда нужна интерпретируемая sparse item–item структура и ранжирование по истории пользователя.
