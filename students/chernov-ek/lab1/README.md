# Лабораторная работа №1. Логическая классификация

В рамках лабораторной работы предстоит реализовать алгоритм построения бинарного решающего дерева и сравнить его с эталонной реализацией.

На лекции были рассмотрены следующие алгоритмы:
* алгоритм построения бинарного решающего дерева ID3;
* алгоритм редукции дерева;
* алгоритм бинаризации вещественного признака;

## Задание

1. выбрать датасет для классификации, например на [kaggle](https://www.kaggle.com/datasets?tags=13302-Classification);
   1. датасет должен содержать пропуски;
   2. датасет должен содержать категориальные и количественные признаки;
2. реализовать алгоритм построения дерева ID3 с критерием Джини;
3. реализовать обработку пропущенных значений через оценку вероятности;
4. обучить дерево на выбранном датасете;
5. оценить качество классификации;
6. реализовать алгоритм редукции дерева;
7. сравнить качество классификации и регрессии до и после редукции дерева;
8. сравнить с [эталонной](https://scikit-learn.org/stable/) реализацией бинарного решающего дерева;
    1. сравнить качество работы;

## Решение

1. Выбран датасет [Diabetes prediction dataset](https://www.kaggle.com/datasets/iammustafatz/diabetes-prediction-dataset) - сборник медицинских и демографических данных от пациентов вместе с их диабетическим статусом. Данные включают такие показатели, как возраст, пол, индекс массы тела (ИМТ), гипертония, сердечно-сосудистые заболевания, история курения, уровень HbA1c и уровень глюкозы в крови. Этот набор данных можно использовать для создания моделей машинного обучения с целью прогнозирования диабета у пациентов на основе их медицинской истории и демографической информации.

Пропуски:
- `gender` - 18/100.000
- `smoking_history` - 35.816/100.000

Так как классы в датасете распределены неравномерно, перед обучением выполняется балансировка обучающей выборки. Сначала применяется oversampling, чтобы увеличить долю миноритарного класса, а затем undersampling, чтобы уменьшить перевес мажоритарного класса. Для смешанных данных используется `SMOTENC`, потому что в наборе есть и числовые, и категориальные признаки. Перед ресемплингом категориальные признаки временно кодируются через `OrdinalEncoder`, а после обработки восстанавливаются в исходный строковый вид. После этого данные передаются в реализацию дерева решений без потери исходной структуры признаков. Итоговое качество модели оценивается на тестовой выборке с помощью `accuracy_score`.

2. Алгоритм построения дерева ID3 с критерием Джини

В работе используется собственная реализация дерева решений `DecisionTreeClassifier`. Дерево строится рекурсивно по схеме ID3: на каждом шаге выбирается признак, который лучше всего разделяет объекты. Качество разбиения оценивается по тому, насколько оно уменьшает неоднородность Джини. Для числовых признаков подбирается лучший порог, а для категориальных создаются ветви по значениям признака. В каждом узле дополнительно сохраняются класс большинства и вероятности классов, чтобы дерево могло корректно работать с пропусками и делать предсказания.

3. Обработка пропущенных значений через оценку вероятности

В режиме `--pass-processing-type weight` значение `Other` в признаке `gender` и значение `No Info` в `smoking_history` рассматриваются как пропуски. При выборе разбиения такие значения не выделяются в отдельную категорию и не создают собственную ветвь. Вместо этого дерево оценивает, как часто объекты переходят в каждую из обычных ветвей, и использует эти вероятности дальше. Во время обучения объект с пропуском не отправляется в одну конкретную ветвь, а распределяется сразу по всем ветвям с соответствующими весами. Во время предсказания итоговый ответ для такого объекта считается как взвешенная комбинация прогнозов из всех подходящих ветвей. Если выбран режим `--pass-processing-type none`, специальные значения обрабатываются как обычные категории без дополнительной логики.

4. Обучение и запуск

После загрузки датасета признаки отделяются от целевой переменной, а затем выборка делится на обучающую и тестовую в соотношении `80/20`. Разбиение выполняется со стратификацией, чтобы сохранить исходное соотношение классов. После подготовки данных модель обучается на тренировочной части, а качество оценивается на тестовой по метрике `accuracy`.

Основные варианты запуска:
- `python source/main.py` запускает программу в базовом режиме, где специальные значения считаются обычными категориями.
- `python source/main.py --pass-processing-type weight` включает обработку пропусков через вероятностное распределение по ветвям.
- `python source/main.py --use-reduction yes` включает редукцию дерева по отдельной валидационной выборке.
- `python source/main.py --show-tree yes` выводит построенное дерево на экран.

5. Качество классификации

- Accuracy без обработки пропусков: **0.9592**
- Accuracy с обработкой пропусков: **0.95695**

6. Алгоритм редукции дерева

Редукция выполняется после полного построения дерева. Если запущен режим `--use-reduction yes`, от обучающей выборки дополнительно отделяется валидационная часть, которая не участвует в построении дерева. Затем дерево обходится снизу вверх: для каждого внутреннего узла проверяется, можно ли заменить всё его поддерево одним листом с классом большинства, не ухудшая accuracy на валидационной выборке. Если качество не падает, поддерево схлопывается, иначе исходная структура узла сохраняется.
