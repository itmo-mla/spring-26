# Лабораторная работа №3. Градиентный бустинг

В рамках данной лабораторной работы предстоит реализовать алгоритм градиентного бустинга и сравнить его с эталонной реализацией из библиотеки `scikit-learn`.

## Задание

1. Выбрать датасет для анализа, например, на [kaggle](https://www.kaggle.com/datasets).
2. Реализовать алгоритм градиентного бустинга.
3. Обучить модель на выбранном датасете.
4. Оценить качество модели с использованием кросс-валидации.
5. Замерить время обучения модели.
6. Сравнить результаты с эталонной реализацией из библиотеки [scikit-learn](https://scikit-learn.org/stable/):
   - точность модели,
   - время обучения.
7. Подготовить отчет, включающий:
   - описание алгоритма градиентного бустинга,
   - описание датасета,
   - результаты экспериментов,
   - сравнение с эталонной реализацией,
   - выводы.

## Решение

1. В качестве данных для обучения выбран датасет [flight_delays_train](https://www.kaggle.com/datasets/kashnitsky/mlcourse/data?select=flight_delays_train.csv).

2. Для решения задачи реализован собственный алгоритм градиентного бустинга для бинарной классификации. В качестве базовых моделей используются неглубокие регрессионные деревья из `sklearn.tree.DecisionTreeRegressor`, которые обучаются на псевдоостатках логистической функции потерь. Итоговый прогноз считается как сумма начальной константы и вкладов всех деревьев, после чего применяется сигмоида.

3. Подготовка данных выполняется в `source/data_loader.py`: целевой признак `dep_delayed_15min` переводится в значения `0` и `1`, а категориальные признаки кодируются через `one-hot` представление. После кодирования получается матрица признаков, пригодная для собственной реализации и для эталонной модели `sklearn`.

4. Эксперимент реализован в `source/main.py`. Для оценки качества используется стратифицированная кросс-валидация на 3 фолдах. Для каждой модели считаются `ROC-AUC`, `Accuracy` и среднее время обучения на одном фолде.

5. Для сравнения выбрана эталонная реализация `sklearn.ensemble.GradientBoostingClassifier` с тем же числом деревьев, шагом обучения и глубиной дерева, что и у собственной модели.

## Отчет

### Описание датасета

Датасет содержит информацию о рейсах и факте задержки вылета более чем на 15 минут. В признаках используются месяц, день месяца, день недели, время вылета, авиакомпания, аэропорт отправления, аэропорт назначения и расстояние перелета. Целевая переменная `dep_delayed_15min` принимает значения `Y` и `N`.

### Описание алгоритма

Градиентный бустинг строит ансамбль моделей последовательно. Сначала модель делает простой константный прогноз, равный логиту доли положительного класса. Затем на каждой итерации считается ошибка текущего ансамбля в виде псевдоостатков `y - p`, где `p` - текущая вероятность задержки. Новое дерево обучается предсказывать эти псевдоостатки, а его прогноз добавляется к ансамблю с коэффициентом `learning_rate`.

### Сравнение с эталонной реализацией

По результатам эксперимента эталонная реализация `sklearn.ensemble.GradientBoostingClassifier` показала более высокое качество классификации. Значение `ROC-AUC` у `sklearn` составило `0.7122 +/- 0.0010`, тогда как у собственной реализации - `0.6967 +/- 0.0016`. По метрике `Accuracy` эталонная модель также оказалась лучше: `0.8142 +/- 0.0004` против `0.8096 +/- 0.0000`.

Время обучения моделей оказалось сопоставимым: собственный бустинг обучался в среднем `37.27` секунды на фолд, а `sklearn GradientBoosting` - `37.63` секунды. Таким образом, основное отличие проявилось не в скорости, а в качестве предсказаний. Это объясняется тем, что реализация `sklearn` использует более проработанные процедуры построения деревьев, оптимизации листьев и обработки численных особенностей, тогда как собственная реализация воспроизводит базовую идею градиентного бустинга в упрощенном виде.

В лабораторной работе реализован простой вариант градиентного бустинга для бинарной классификации и подготовлен единый эксперимент для сравнения с `sklearn`. По результатам запуска эталонная модель показала более высокие значения `ROC-AUC` и `Accuracy`, а время обучения обеих моделей оказалось сопоставимым.
