# Лабораторная работа 5: SLIM и латентная семантическая модель (NNMF)

## 1. Датасет

Использован текстовый корпус **20 Newsgroups** из `sklearn.datasets`:

- категории: `sci.med`, `sci.space`;
- тексты преобразованы в разреженную матрицу **документ–термин** с помощью `TfidfVectorizer` (до 300 терминов);
- документы — «пользователи», термины — «объекты», веса TF-IDF — «рейтинги»;
- 20% ненулевых элементов матрицы отложены в тест.

| Параметр | Значение |
|----------|----------|
| Документов | 1977 |
| Терминов | 300 |
| Элементов в train | 63268 |
| Элементов в test | 15816 |

## 2. SLIM (Sparse Linear Methods)

**Идея:** \(\hat{R} = R \cdot W\), где \(W\) — разреженная матрица весов между объектами (elastic-net по столбцам).

| Реализация | Файл                                                                            |
|------------|---------------------------------------------------------------------------------|
| Своя | `source/slim.py` — `SlimModel`                                                  |
| Библиотечная | `source/references.py` — `LibrarySlimModel` (`sklearn.linear_model.ElasticNet`) |

## 3. Выбор латентной семантической модели

Рассматривались три варианта:

| Модель | Суть | Почему не выбрана |
|--------|------|-------------------|
| **LFM** (Latent Factor Model) | \(R \approx U V^T\), факторы могут быть отрицательными | TF-IDF неотрицателен; отрицательные факторы хуже интерпретируются для текста |
| **ALS** (Alternating Least Squares) | Чередующиеся наименьшие квадраты для факторизации | Ориентирован на collaborative filtering; для разреженной TF-IDF матрицы NNMF естественнее |
| **NNMF** (Non-Negative Matrix Factorization) | \(R \approx W H\), \(W, H \geq 0\) | **Выбрана:** веса TF-IDF \(\geq 0\), латентные темы интерпретируемы как неотрицательные комбинации терминов |

## 4. NNMF (Non-Negative Matrix Factorization)

**Идея:** документ–термин матрица раскладывается на две неотрицательные матрицы:

- W — латентные представления документов;
- H — латентные представления терминов.

| Реализация | Файл                                       | Алгоритм |
|------------|--------------------------------------------|----------|
| Своя | `source/nmf.py` — `NmfModel`               | `sklearn.decomposition.NMF`, solver=`mu` |
| Библиотечная | `source/references.py` — `LibraryNmfModel` | `sklearn.decomposition.NMF`, solver=`cd` |

## 5. Метрики (sklearn)

- **RMSE** — `sklearn.metrics.root_mean_squared_error`
- **NDCG@10** — `sklearn.metrics.ndcg_score`

## 6. Результаты

Параметры: `max_features=300`, `test_size=0.2`, SLIM: `l1=0.5`, `l2=1.0`, NNMF: `n_components=40`.

| Модель | RMSE (своя) | RMSE (библ.) | NDCG@10 (своя) | NDCG@10 (библ.) |
|--------|-------------|--------------|----------------|-----------------|
| SLIM   | 0.1544      | 0.1544       | 0.0149         | 0.0149          |
| NNMF   | 0.1447      | 0.1455       | 0.0132         | 0.0125          |

## 7. Выводы

1. Для TF-IDF матрицы **NNMF** предпочтительнее LFM и ALS: неотрицательность данных и интерпретируемость тем.
2. SLIM моделирует локальные связи между терминами; NNMF — глобальные латентные темы.
3. Сравнение своих и библиотечных реализаций выполняется единообразно через метрики Cornac.
