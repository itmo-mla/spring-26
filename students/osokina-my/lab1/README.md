# Random Forest: реализация и эксперименты

## Описание метода

**Random Forest** (Случайный лес) — метод ансамблевого обучения, объединяющий множество деревьев решений в один классификатор. В данной реализации используются:

1. **Bootstrap-агрегирование (Bagging)** — каждое дерево обучается на случайной выборке с возвращением из исходной обучающей выборки. Объекты, не попавшие в выборку, образуют **OOB (Out-of-Bag)** — тестовую подвыборку для оценки без отдельной валидации.

2. **Метод случайных подпространств (RSM)** — при построении каждого узла дерева рассматривается только случайное подмножество признаков (обычно √p или log₂(p), где p — число признаков). Это снижает корреляцию между деревьями и повышает обобщающую способность.

3. **OOB-оценка** — агрегированное предсказание по тем деревьям, где объект не участвовал в обучении.

4. **Важность признаков OOB^j** — для признака j выполняется пермутация его значений в OOB-выборке; важность измеряется как падение OOB-точности. Чем сильнее падение, тем важнее признак.

## Датасет

**Wine** — классический датасет UCI для задачи классификации:
- **Объекты:** 178 образцов вина
- **Признаки:** 13 (химический состав: алкоголь, кислотность, зольность, магний, фенолы, флаваноиды и др.)
- **Классы:** 3 сорта винограда (культивары)

Разбиение: 70% обучение, 30% тест (стратифицированная выборка).

## Структура проекта

Основной алгоритм:
- `CustomRandomForestClassifier` — Реализация Random Forest с использованием sklearn DecisionTreeClassifier
как базового алгоритма. Включает метод случайных подпространств (RSM)
и оценку OOB.

Утилитарные методы (source/utils.py): 
- `train_test_split_bootstrap(X, y)` — bootstrap-выборка и OOB-индексы
- `get_random_subspace_indices(n_features, max_features)` — выбор случайного подпространства признаков (RSM)
- `oob_score(X, y, trees, oob_indices_per_tree)` — оценка точности по OOB
- `permutation_importance_oob(...)` — важность признаков через пермутацию OOB (OOB^j)

## Результаты экспериментов

### Grid Search по OOB

Сетка параметров:
- `n_estimators`: [50, 100, 150]
- `max_features`: ['sqrt', 'log2', 0.5]
- `max_depth`: [5, 10, 15, None]
- `min_samples_split`: [2, 5]

Оптимальные параметры подбираются по максимальному OOB Score. Лучшие параметры (по OOB): {
    'max_depth': 5, 
    'max_features': 'sqrt', 
    'min_samples_split': 2, 
    'n_estimators': 100, 
    'random_state': 42
    }
