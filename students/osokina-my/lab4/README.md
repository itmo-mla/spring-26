## Наивный байесовский классификатор

**Наивный байесовский классификатор (Naive Bayes)** — генеративная модель для задачи классификации. Предполагается, что объекты x порождаются классом y , а признаки при фиксированном классе **независимы** (наивное предположение).

Для непрерывных признаков часто берут гауссовское распределение. Обучение сводится к оценке априорных вероятностей классов p(y) и параметров распределений признаков по обучающей выборке (метод максимального правдоподобия).

**Связь с GMM:** и Naive Bayes, и GMM относятся к **генеративным** моделям, оценивающим совместное распределение данных. Naive Bayes использует дискретную скрытую переменную «класс» и диагональную (факторизованную) структуру ковариации. GMM — это смесь $K$ многомерных гауссиан с **полной** ковариацией; скрытая переменная — индекс компоненты смеси. Обе модели обучаются через максимизацию правдоподобия; для GMM с неизвестными принадлежностями к компонентам применяют алгоритм **EM**.

**Алгоритм EM:**
- **E-шаг:** вычисление апостериорных весов, то есть в какой степени каждый объект выборки принадлежит j-ой компоненте.
- **M-шаг:** обновление по взвешенной выборке, максимизация логарифма правдоподобия для каждой компоненты отдельно.
- Критерий сходимости: остановка, когда увеличение логарифма правдоподобия на текущей итерации становится меньше заданного порога `tol`.

## Сравнение с эталонной реализацией (sklearn)

1. **Логарифм правдоподобия** — основной критерий ML. При K=3 и K=5 собственная реализация даёт **не ниже** sklearn (разница среднего score до 0.3 и 1.7 соответственно). При K=2 sklearn выше — EM чувствителен к локальным оптимумам и разной инициализации.
2. **Число итераций EM** различается (например, при K=3: 60 vs 24) при сходных значениях правдоподобия — допустимо при разных траекториях оптимизации.
3. **AIC/BIC** — sklearn при K=3 предпочтительнее по AIC; при K=5 собственная модель сильно лучше по обоим критериям из-за более высокого LL.
4. Реализации согласованы по формулам; расхождения объясняются **стохастической инициализацией** и локальными максимумами EM, а не ошибкой в формулах E/M-шагов.

## Выводы

1. Реализованная GMM корректно максимизирует логарифм правдоподобия и восстанавливает многомерную плотность на Wine.
2. При согласованной случайной инициализации результаты **близки к sklearn**; для K=3 отличие среднего log-likelihood составляет около 0.3.
3. Увеличение K монотонно улучшает правдоподобие на обучающей выборке; критерии AIC/BIC помогают выбрать разумный компромисс сложности.
4. Кластеры GMM частично соответствуют сортам винограда (ARI ≈ 0.3–0.4), что подтверждает наличие группировки в данных, хотя модель обучалась **без меток**.
5. GMM обобщает идеи генеративного моделирования, общие с наивным байесом, но позволяет моделировать корреляции признаков через полную ковариационную матрицу каждой компоненты.
