# Лабораторная работа 3: стохастический градиентный бустинг

## Описание алгоритма градиентного бустинга

**Градиентный бустинг** представляет собой приближение бустинга с использованием градиента функции потерь. 
В отличие от AdaBoost, он работает с произвольной дифференцируемой функцией потерь, а не только с экспоненциальной. 
В частности это позволяет решать не только задачу классификации, но и регрессии.

В качестве базовых моделей чаще всего используются решающие деревья небольшой глубины (gradient boosting over decision trees, GBDT).

**Стохастический** вариант (Stochastic Gradient Boosting) на каждой итерации строит дерево только на
**случайной подвыборке** объектов (доля выборки `subsample`, в эксперименте 0.7, то есть 70%). 
Это снижает корреляцию между деревьями, ускоряет обучение и улучшает обобщение.

Реализация: модуль `sgb.py` (`StochasticGradientBoostingClassifier`). В качестве базового алгоритма
используется `DecisionTreeRegressor` из scikit-learn; сам цикл бустинга, сабсэмплинг и логистическая постановка реализованы вручную.

## Описание датасета

Используется встроенный датасет **Breast Cancer Wisconsin (diagnostic)** из `sklearn.datasets.load_breast_cancer`:

- **Задача:** бинарная классификация — доброкачественная (B) vs злокачественная (M) опухоль.
- **Объёмы:** 569 объектов, 30 числовых признаков (статистики ядер клеток на изображениях).
- **Предобработка:** признаки стандартизованы (`StandardScaler`), чтобы сравнение с эталоном не зависело от масштаба.

## Результаты экспериментов

Запуск: ноутбук `main.ipynb` (кросс-валидация и замер времени).

**Настройки:** `StratifiedKFold`, 5 фолдов, `shuffle=True`, `random_state=42`; `n_estimators=100`, `learning_rate=0.1`,
`max_depth=3`, `subsample=0.7`.

Пример результатов:

| Метрика | Своя реализация | sklearn `GradientBoostingClassifier` |
|--------|------------------|--------------------------------------|
| Средняя accuracy (5-fold CV) | **0.951** | **0.960** |
| Ст. откл. по фолдам | 0.024 | 0.015 |
| Среднее время одного `fit` на полной выборке (5 прогонов) | **~0.37 с** | **~0.40 с** |

## Сравнение с эталонной реализацией

- **Точность:** эталонный `GradientBoostingClassifier` с `loss='log_loss'` и теми же гиперпараметрами показал несколько
**более высокую** среднюю accuracy на кросс-валидации. Это ожидаемо: в sklearn реализованы детали оптимизации дерева,
критерии остановки и численная устойчивость.
- **Время:** собственная реализация в тестовом прогоне оказалась **сопоставима или чуть быстрее**.

## Выводы

1. Стохастический градиентный бустинг с подвыборкой ~70% на каждом дереве успешно обучается на датасете диагностирования рака и даёт высокую точность (~95% accuracy в CV).
2. Идея метода — последовательное добавление деревьев, обученных на градиенте логистической потери, с сэмплированием — реализация в `sgb.py`.
3. Эталон sklearn даёт **лучшую** среднюю CV-точность при тех же основных гиперпараметрах; **время обучения** одного прогона — того же порядка.
4. Для воспроизведения: `pip install -r requirements.txt`, затем открыть и выполнить `main.ipynb`.
