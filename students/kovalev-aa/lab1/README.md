# Отчет по лабораторной работе: Реализация и анализ дерева решений с критерием Джини

## Цель работы
- Изучение алгоритма построения дерева решений ID3 с использованием критерия Джини.
- Практическая реализация обработки категориальных и числовых признаков с пропущенными значениями.
- Реализация редукции (pruning) дерева и оценка влияния на качество классификации.
- Сравнение собственной реализации дерева с эталонной реализацией `sklearn`.

## Задачи
1. Выбрать датасет для задачи классификации (например, Adult с OpenML).
2. Убедиться, что датасет содержит:
   - Пропущенные значения;
   - Категориальные и количественные признаки.
3. Реализовать алгоритм построения дерева решений ID3 с критерием Джини.
4. Обработать пропущенные значения через оценку вероятности (категориальные) и среднее значение (числовые).
5. Обучить дерево на выбранном датасете.
6. Оценить качество классификации на тестовой выборке.
7. Реализовать алгоритм редукции дерева (post-pruning).
8. Сравнить качество классификации до и после редукции.
9. Сравнить результаты с реализацией `DecisionTreeClassifier` из `sklearn`.

## Используемые инструменты
- Python 3.12
- Библиотеки: `pandas`, `numpy`, `sklearn`
- Датасет: Adult (OpenML), с пропусками, категориальными и числовыми признаками.

## Реализация

### Подготовка датасета
- Категориальные признаки закодированы через one-hot encoding.
- Пропуски:
  - Числовые признаки → заполнение средним значением;
  - Категориальные признаки → случайный выбор на основе распределения.

### Алгоритм дерева решений
- Критерий разделения: Gini impurity.
- Рекурсивная постройка дерева до тех пор, пока:
  - Все примеры принадлежат одному классу, или
  - Остались признаки для разделения, или
  - Достигнута максимальная глубина.
- Пост-обрезка (post-pruning) с использованием валидационного набора и параметра `ccp_alpha`.

### Оценка качества
- Метрики: precision, recall, f1-score, accuracy.
- Сравнение для:
  - Собственное дерево (`OWN`)
  - Собственное дерево с редукцией (`OWN PRUNING`)
  - `sklearn DecisionTreeClassifier` (`SKLEARN`)
  - `sklearn` с редукцией (`SKLEARN PRUNING`)

## Выводы
1. Реализация дерева решений ID3 с критерием Джини и обработкой пропущенных значений показала стабильную работу.
2. Прунинг на данном датасете практически не изменил метрики, что указывает на низкий уровень переобучения исходного дерева.
3. Собственная реализация дерева близка к работе `sklearn` по качеству классификации.
4. Метрики класса 1 (миноритарного класса) ниже из-за дисбаланса классов в датасете.
5. Для улучшения классификации миноритарного класса рекомендуется балансировка выборки или использование методов взвешивания классов.

## Заключение
- Задача классификации решена с использованием собственного алгоритма дерева решений и стандартного `sklearn`.
- Прунинг показал небольшое влияние, что подтверждает корректную работу построения дерева.
- Работа демонстрирует базовую методологию подготовки данных, построения, редукции и оценки дерева решений в задачах классификации.
