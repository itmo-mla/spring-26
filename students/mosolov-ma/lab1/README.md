# Лабораторная работа №1. Логическая классификация

## Датасет

Использован датасет **Heart Attack Prediction** из Kaggle.
Содержит 303 образца, 13 признаков (возраст, пол, холестерин, давление и др.) и целевую переменную — наличие/отсутствие сердечного приступа.

Датасет загружается автоматически через `kagglehub` при запуске.

## Пропуски

В исходном датасете пропусков не было, поэтому они добавлены искусственно:
- 10 % значений в обучающей и тестовой выборках случайным образом заменены на `None`
- При предсказании пропуски обрабатываются через **взвешенное распределение классов** по обеим ветвям дерева

## Описание реализации

### Дерево решений (tree.py)

- **Критерий разбиения** — индекс Джини (также доступна энтропия)
- **Построение** — рекурсивный алгоритм с предобрезкой (`max_depth`, `min_samples_split`, `min_gain`)
- **Пропуски** — при достижении узла с `None` значением предсказание вычисляется как взвешенная сумма распределений левой и правой ветвей
- **Усечение** — алгоритм Reduced Error Pruning (REP): узел заменяется на лист, если ошибка на валидационной выборке не ухудшается

### Сравнение со sklearn

Для сравнения использован `DecisionTreeClassifier` из `sklearn` с теми же гиперпараметрами.

## Результаты

| Параметр               | Custom  | Sklearn |
|------------------------|---------|---------|
| Узлов решения          | ~10-15  | ~10-15  |
| Листьев                | ~12-18  | ~12-18  |
| Глубина                | ≤5      | ≤5      |
| Accuracy (test)        | ~0.78   | ~0.78   |
| F1-score (test)        | ~0.75   | ~0.75   |

### Выводы

- Качество собственной реализации **сопоставимо** с эталонной реализацией из sklearn
- Редукция дерева уменьшает число узлов, но на данном датасете не даёт значительного прироста качества
- Обработка пропусков через взвешенное распределение работает корректно и позволяет делать предсказания на данных с пропусками
