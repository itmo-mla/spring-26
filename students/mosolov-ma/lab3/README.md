# Лабораторная работа №3. Градиентный бустинг

## Цель работы

Реализовать алгоритм градиентного бустинга, обучить его на датасете по предсказанию риска сердечного приступа и сравнить результат с `scikit-learn`.

## Теория

Градиентный бустинг:

`F_m(x) = F_prev(x) + alpha_m * b_m(x)`

где новый базовый алгоритм `b_m` обучается на антиградиенте функции потерь. Для бинарной классификации здесь использована логистическая функция потерь:

`L(y, F) = log(1 + exp(-y * F)), y in [-1, +1]`

На каждом шаге:

1. вычисляются псевдо-остатки, то есть отрицательный градиент;
2. обучается готовое регрессионное дерево `DecisionTreeRegressor` из `sklearn` небольшой глубины;
3. выполняется одномерная минимизация шага `alpha_m`;
4. ансамбль обновляется с коэффициентом `learning_rate`.

Также реализован стохастический вариант бустинга: на каждом шаге используется случайная подвыборка объектов.

## Датасет

Использован датасет Kaggle:

`nareshbhat/health-care-data-set-on-heart-attack-possibility/heart.csv`

- объектов: 303;
- признаков: 13;
- целевая переменная: `target`;
- доля положительного класса: 0.545.

Признаки уже числовые, пропусков в датасете нет, поэтому дополнительное кодирование не требуется.

## Параметры модели

- число деревьев: 120;
- learning rate: 0.1;
- максимальная глубина дерева: 3;
- min_samples_leaf: 5;
- subsample: 0.8 для стохастического бустинга.

## Выводы

1. Реализован градиентный бустинг: последовательное добавление базовых моделей по антиградиенту функции потерь, при этом базовая модель берется из `sklearn`.
2. Собственная реализация и `scikit-learn` обучаются на одном и том же датасете и оцениваются одинаковыми метриками.
3. Стохастический режим позволяет уменьшить время обучения.
4. Полученные результаты показывают, что реализованный бустинг работает сопоставимо с эталонной реализацией на маленьком табличном датасете.
