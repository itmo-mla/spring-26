# Лабораторная работа №2. Ансамбли моделей — Random Forest

## Описание метода

**Random Forest** — ансамблевый метод, сочетающий бэггинг и случайные подпространства (RSM). Для каждого дерева генерируется bootstrap-выборка, а при построении дерева на каждом шаге рассматривается случайное подмножество признаков размера `max_features`. Итоговый прогноз — голосование всех деревьев.

В моей реализации `max_features` передается в `DecisionTreeClassifier`, то есть случайное подмножество признаков выбирается на каждом разбиении дерева, как в `sklearn`. Дополнительно после обучения выполняется OOB-отбор деревьев: в ансамбль попадают только те базовые модели, чье качество на собственных OOB-объектах не ниже среднего.

**OOB (Out-Of-Bag) оценка** — ~37% данных, не вошедших в bootstrap-выборку, используются для оценки качества без отдельной валидационной выборки.

**Важность признаков OOB^j** — permutation-based: значения признака `j` перемешиваются в OOB-образцах, измеряется падение accuracy.

## Описание датасета

Датасет «Health Care Data Set on Heart Attack Possibility» (Kaggle). Файл `heart.csv` содержит 303 записи и 13 признаков для предсказания риска сердечного приступа (`target` — бинарная целевая переменная).

Признаки: возраст, пол, тип боли в груди (`cp`), давление (`trestbps`), холестерин (`chol`), уровень сахара (`fbs`), ЭКГ (`restecg`), максимальная частота сердцебиения (`thalach`), стенокардия (`exang`), депрессия ST (`oldpeak`), наклон ST (`slope`), число окрашенных сосудов (`ca`), талассемия (`thal`).

## Результаты экспериментов

Подбор гиперпараметров через Grid Search по OOB:
- `n_estimators`: [50, 100, 150]
- `max_depth`: [None, 5, 10]
- `max_features`: ['sqrt', 'log2']
- `min_samples_split`: [2, 5]

**Лучшие параметры:** `max_depth=None`, `max_features='sqrt'`, `min_samples_split=5`, `n_estimators=150`

## Выводы

1. **OOB score у custom выше** (0.8515 vs 0.8251) — после введения OOB-отбора базовых деревьев моя реализация стала лучше по собственной OOB-оценке, чем стандартный `sklearn`-ансамбль на тех же гиперпараметрах.

2. **Accuracy на обучающей выборке** тоже чуть выше у custom-реализации (0.9868 vs 0.9835), что ожидаемо при OOB-отборе деревьев: ансамбль стал немного более “подобранным” под обучающие данные.

3. **Время обучения** у custom-реализации стало чуть больше (0.3170 vs 0.2791 сек), потому что дополнительно считается OOB-качество каждого дерева и выполняется отбор. На больших данных `sklearn` по-прежнему должен выигрывать за счёт Cython-оптимизаций и параллелизма.

4. **Важность признаков** — три разных метода дают разные абсолютные значения, но согласованы в ранжировании топ-признаков:
   - `ca`, `cp`, `oldpeak` и `exang` — наиболее заметные признаки в custom-оценке и в permutation-оценке `sklearn`.
   - `sex` и `thalach` — практически не влияют на качество при перемешивании.
   - Значения `impurity` и permutation/OOB-оценок остаются различными по масштабу, но дают близкое ранжирование ключевых признаков.

5. **Разница методов оценки важности:**
   - **OOB^j (permutation)** — измеряет вклад признака через падение accuracy при его перемешивании, может быть отрицательным, не нормирован.
   - **sklearn impurity** — сумма уменьшений Gini по всем разбиениям, всегда положительна, нормирована в сумму 1, но склонна завышать признаки с большим числом уникальных значений.
   - **sklearn permutation** — аналог OOB^j, но на полной выборке с многократной перестановкой; значения ближе к custom, но абсолютные величины другие из-за использования всех данных вместо OOB.
