## Отчёт

### 1. Naive Bayes и связь с GMM

Naive Bayes — классификатор по теореме Байеса, с предположением, что признаки независимы при фиксированном классе.  
Тогда `p(x|C_k)` раскладывается в произведение одномерных плотностей, и модель проще оценивать.

GMM тоже использует Байеса, но на E-шаге EM: считаются ответственности `γ(z_ik)` — насколько точка «принадлежит» компоненте k.  
Отличие: NB — обучение с учителем и диагональные ковариации; GMM — без меток и полные ковариации, можно ловить корреляции признаков.

### 2. Датасет

**Wine** (`sklearn.datasets.load_wine`): 178 образцов, 13 признаков, 3 сорта винограда (59 / 71 / 48).

Перед обучением — `StandardScaler`. Для графиков — PCA на 2 компоненты (~55% дисперсии).

### 3. Реализация (кратко)

Модель: `p(x|θ) = Σ_k π_k · N(x|μ_k, Σ_k)`, обучение EM.

- **Старт:** k-means++ (свой код на NumPy), из кластеров берутся `μ_k`, `Σ_k`, `π_k`.
- **E-шаг:** логи весов и `log N(x|μ,Σ)` через Cholesky; нормировка через `logsumexp`.
- **M-шаг:** взвешенные средние и ковариации; к `Σ_k` добавляется `1e-4·I`.
- **Остановка:** если прирост log-likelihood < `1e-6`, иначе до 300 итераций.
- **n_init=10:** несколько запусков, лучший по log-likelihood среди решений без «пустых» компонент.

Сравнение с sklearn: `GaussianMixture`, `covariance_type='full'`, те же `max_iter`, `tol`, `n_init`, `random_state=42`.

### 4. Результаты (K = 3)

| | самописная GMM | sklearn |
|---|---|---|
| log-likelihood (сумма) | −2068.03 | −2068.03 |
| AIC | 4764.07 | 4764.06 |
| BIC | 5763.15 | 5763.14 |
| ARI | 0.9471 | 0.9471 |
| Silhouette | 0.279 | 0.279 |
| итераций EM | 28 | 21 |

Кластеры совпадают (MSE ответственностей ≈ 0). Регуляризация ковариаций: `1e-4` у самописной, `1e-6` у sklearn.

**BIC по K (1…7):** минимум у самописной модели при **K = 2**; у sklearn тоже минимум при K = 2.

| K | BIC (самопис.) | BIC (sklearn) |
|---|----------------|---------------|
| 1 | 5741 | 5741 |
| 2 | **5721** | **5630** |
| 3 | 5773 | 5773 |

### 6. Выводы

Самописная GMM даёт те же кластеры, что sklearn (ARI ≈ 0.95). Важны `logsumexp` и регуляризация `Σ_k`.

По BIC для Wine выгоднее **K=2**, хотя в данных три класса. При K=3 кластеризация всё равно хорошая, просто BIC штрафует лишние параметры.
