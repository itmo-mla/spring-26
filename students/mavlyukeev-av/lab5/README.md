# Лабораторная работа №5. Рекомендательные системы

## SLIM

**SLIM** (Sparse Linear Methods) — item-based метод для Top-N рекомендаций. Для каждого товара ($$j$$) подбирается разреженный вектор весов ($$\mathbf{w}_j$$), чтобы столбец рейтингов \( $$\mathbf{r}_j$$ ) был близок к линейной комбинации остальных товаров:

$$
\min_{\mathbf{w}_j} \;\frac{1}{2}\|\mathbf{r}_j - R\mathbf{w}_j\|_2^2 + \lambda_1\|\mathbf{w}_j\|_1 + \frac{\lambda_2}{2}\|\mathbf{w}_j\|_2^2,
\quad w_{jj} = 0,\; w_{ij} \ge 0.
$$

L1 даёт разреженность, L2 — устойчивость. Предсказание: ($$\hat{r}_{uj} = \mathbf{r}_u^\top W_{:,j}$$), где ($$R$$) — обучающая матрица взаимодействий, ($$W$$) — веса SLIM.

В `slim.py` класс `SLIM` обучается координатным спуском для Elastic Net, без вызова sklearn внутри модели.

## LSA

В качестве латентной семантической модели взята **LSA** (Latent Semantic Analysis). Тексты отзывов по товару склеиваются в один документ, по ним строится TF-IDF (товар × термин), затем TruncatedSVD выделяет латентные темы. Профиль пользователя — среднее по латентным векторам товаров, которые он оценил. Предсказание рейтинга:

$$
\hat{r}_{ui} = \mu + \mathbf{p}_u^\top \mathbf{q}_i,
$$

($$\mu$$) — средний рейтинг на train, ($$\mathbf{p}_u$$), ($$\mathbf{q}_i$$) — факторы пользователя и товара.

Эталон в ноутбуке: `TfidfVectorizer` + `TruncatedSVD` из sklearn, та же схема.

## Датасет

Использован [Amazon Fine Food Reviews](https://www.kaggle.com/datasets/snap/amazon-fine-food-reviews) (`snap/amazon-fine-food-reviews`): отзывы с оценкой 1–5 и полем `Text`. Загрузка через `kagglehub`, код — в `source/lab5.ipynb`.

| Поле | Смысл |
|------|--------|
| `UserId`, `ProductId` | пользователь и товар |
| `Score` | оценка |
| `Text` | текст отзыва |

Пайплайн: отбор активных пользователей, отсечение редких товаров, split 80/20, для SLIM оставлены 100 самых частых товаров (иначе обучение слишком долгое). Итоговая матрица на train: **865 × 100**, 1085 оценок в обучении, 226 пар в тесте.

## Результаты

Считал в `run_experiments.py` и в ноутбуке — настройки совпадают.

| Модель | RMSE | NDCG@10 |
|--------|------|---------|
| SLIM, самописная | **3.606** | 0.647 |
| SLIM, эталон (ElasticNet) | 3.947 | 0.652 |
| LSA, самописная | **1.122** | 0.652 |
| LSA, эталон (sklearn) | 1.122 | 0.652 |

**RMSE** — по парам (user, item) из теста.  
**NDCG@10** (*): товар релевантен, если рейтинг ≥ 4; ранжирование по предсказанному скору внутри тестовых позиций пользователя.

## Сравнение с эталоном

**SLIM.** Эталон — `ElasticNet` из sklearn на каждый товар (`positive=True`, `l1_ratio=0.1`, `alpha=1.0`, top-50 весов). Самописный вариант — coordinate descent с `l1=1`, `l2=1`, `top_k=50`. RMSE отличаются примерно на 0.34, у самописной чуть ниже на этом срезе; NDCG почти одинаковый. Разница ожидаема: другой солвер (CD против ElasticNet) и подвыборка данных.

**LSA.** Самописная и эталонная схемы совпадают (TF-IDF → SVD → профиль пользователя), метрики совпали полностью.

## Выводы

1. На разреженной матрице Amazon SLIM даёт RMSE около 3.6–3.9 — для абсолютных оценок при малой плотности это слабовато; для Top-N по implicit feedback метод обычно уместнее.
2. LSA с текстом отзывов заметно лучше по RMSE (~1.12): текст частично компенсирует редкие коллаборативные связи.
3. Самописный SLIM по порядку величин совпадает с ElasticNet-SLIM; самописная LSA повторяет sklearn.
4. NDCG@10 у всех моделей около 0.65 — ранжирование «хороших» отзывов (≥ 4) сопоставимо.
