# Лабораторная 2: Random Forest

## Метод

Реализован классификатор Random Forest:

- каждое дерево обучается на bootstrap-выборке с возвращением;
- качество подбирается по OOB-оценке;
- в узлах дерева случайно выбирается подмножество признаков (`max_features`);
- итоговый прогноз строится голосованием деревьев;
- важность признаков считается через OOB permutation importance: падение OOB accuracy после перемешивания признака.

## Датасет

Использован датасет Titanic:

- объектов: 891;
- train/test: 712 / 179;
- целевая переменная: `Survived`;
- признаки: `Pclass`, `Sex`, `Age`, `SibSp`, `Parch`, `Fare`, `Embarked`;
- пропуски заполнены по train-части: числовые признаки медианой, категориальные - самым частым значением.

## Эксперименты

Гиперпараметры подбирались grid search по OOB accuracy.


| Модель                         | Лучшие параметры                                                               | OOB accuracy | Test accuracy | Grid search, сек | Fit, сек |
| ------------------------------ | ------------------------------------------------------------------------------ | ------------ | ------------- | ---------------- | -------- |
| Custom Random Forest           | `max_depth=5`, `max_features=None`, `min_samples_split=2`, `n_estimators=30`   | 0.8216       | 0.8156        | 13.644           | 2.028    |
| sklearn RandomForestClassifier | `max_depth=3`, `max_features='sqrt'`, `min_samples_split=2`, `n_estimators=30` | 0.8272       | 0.7989        | 0.526            | 0.048    |


## Важность признаков

| ---------- | -------- |

| `Embarked` | 0.0014   |


## Выводы

Собственная реализация показала качество на тестовой выборке немного выше эталонной реализации sklearn на выбранной сетке параметров: 0.8156 против 0.7989. При этом sklearn значительно быстрее: финальное обучение заняло 0.048 сек против 2.028 сек.
