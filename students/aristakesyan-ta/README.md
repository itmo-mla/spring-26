# Лабораторная работа №5. Рекомендательные системы

## Описание SLIM

SLIM (Sparse Linear Method) строит item-item матрицу похожести `W`. Для каждого объекта `j` решается ElasticNet-регрессия:

```text
R_j ~= R * W_j
```

Ограничения:

- диагональ `W_jj = 0`, чтобы объект не предсказывал сам себя;
- веса неотрицательные;
- L1-регуляризация делает матрицу похожести разреженной;
- L2-регуляризация стабилизирует решение.

В работе сравниваются две SLIM-постановки: собственная explicit-rating адаптация обучает регрессию только на пользователях, реально оценивших целевой фильм, а эталонная KarypisLab-style постановка обучается на всей разреженной матрице. ElasticNet-солвер взят из `sklearn`, что допустимо по условиям задания.

Эталонной реализацией SLIM считается KarypisLab SLIM.

## Описание латентной модели

В качестве латентной семантической модели реализована FunkSVD-факторизация:

```text
r_ui = global_mean + user_bias_u + item_bias_i + p_u * q_i
```

Параметры обучаются стохастическим градиентным спуском по известным рейтингам. Модель восстанавливает скрытые предпочтения пользователей и скрытые признаки фильмов.

Эталон для сравнения латентной модели — `sklearn.decomposition.TruncatedSVD`.

## Описание датасета

Использован MovieLens 100K от GroupLens.

- Полный датасет: `100000` оценок.
- Шкала оценок: от `1` до `5`.
- В эксперименте оставлены `300` самых активных пользователей и `300` самых популярных фильмов, чтобы SLIM обучался быстро.
- После фильтрации: матрица `300 x 300`.
- Обучающая выборка: `30566` оценок.
- Тестовая выборка: `7644` оценки.

Разбиение выполнялось по пользователям: у каждого пользователя часть оценок переносилась в тест, минимум одна оценка оставалась в train.

## Результаты экспериментов

Метрики:

- `RMSE` — ошибка восстановления рейтингов, меньше лучше;
- `NDCG@10` — качество ранжирования отложенных test-items пользователя, больше лучше.


| Модель                 | Reference                              | RMSE     | NDCG@10  |
| ---------------------- | -------------------------------------- | -------- | -------- |
| SLIM                   | собственная реализация | `1.1903` | `0.8759` |
| KarypisLab SLIM        |  эталонная постановка  | `2.2463` | `0.8703` |
| FunkSVD                | собственная реализация                 | `0.8868` | `0.8924` |
| sklearn TruncatedSVD | эталон для латентной модели            | `0.9855` | `0.8611` |


Для собственной SLIM-версии разреженность матрицы весов получилась около `0.833`, для KarypisLab-style варианта — около `0.884`.

## Сравнение с эталонными реализациями

FunkSVD по `RMSE` оказался лучше `TruncatedSVD`, потому что обучается напрямую на известных рейтингах и учитывает пользовательские и объектные смещения.

Собственная SLIM-адаптация дала меньший `RMSE`, чем KarypisLab-style вариант. Эталонная KarypisLab-style постановка хуже по `RMSE`, так как восстанавливает всю разреженную матрицу, включая нули неизвестных оценок, но по `NDCG@10` обе SLIM-версии близки.

## Выводы

В ходе работы были реализованы две рекомендательные модели: SLIM и FunkSVD. На MovieLens 100K факторизационная модель дала лучший `RMSE` и лучший `NDCG@10` среди реализованных моделей.

Сравнение с `TruncatedSVD` подтвердило, что обучение факторизации непосредственно по рейтингам дает преимущество в восстановлении оценок. SLIM оказался полезен для ранжирования, но его высокий `RMSE` показывает, что такая постановка хуже подходит для предсказания точных explicit-рейтингов.
