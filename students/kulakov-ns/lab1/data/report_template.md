# Лабораторная работа № 1

## Цель работы
Реализовать собственное бинарное дерево решений с критерием Джини, обработкой пропусков через вероятностную оценку и редукцией дерева, затем сравнить качество модели до и после редукции, а также с эталонной реализацией `DecisionTreeClassifier` из `sklearn`.

## Выбранный датасет
Для работы использован датасет [Titanic](https://www.kaggle.com/competitions/titanic/overview).

Причины выбора:
- есть пропуски (`Age`, `Embarked`, `Cabin`);
- есть количественные признаки (`Age`, `SibSp`, `Parch`, `Fare`);
- есть категориальные признаки (`Pclass`, `Sex`, `Embarked`, `CabinDeck`);
- задача является бинарной классификацией (`Survived`).

## Подготовка данных
Из исходного датасета использовались признаки:
`Pclass`, `Sex`, `Age`, `SibSp`, `Parch`, `Fare`, `Embarked`, `CabinDeck`.

Признак `CabinDeck` был получен из поля `Cabin` как первая буква палубы.

## Вывод
В ходе работы было реализовано собственное бинарное дерево решений с критерием Джини и вероятностной обработкой пропусков. После редукции дерево стало компактнее, а качество классификации на тестовой выборке немного улучшилось. Сравнение с `sklearn` показало, что собственная реализация работает корректно, но эталонная библиотечная модель обычно оказывается стабильнее и удобнее в использовании.
