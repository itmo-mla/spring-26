# Лабораторная работа № 3

## Цель работы
Реализовать алгоритм градиентного бустинга, обучить модель на датасете Titanic, выполнить оценку качества с использованием кросс-валидации и сравнить результат с эталонной реализацией `GradientBoostingClassifier` из `scikit-learn`.

## Выбранный датасет
Для работы использован датасет [Titanic](https://www.kaggle.com/competitions/titanic/overview).

## Подготовка данных
Использованы признаки:
`Pclass`, `Sex`, `Embarked`, `CabinDeck`, `Age`, `SibSp`, `Parch`, `Fare`.

Признак `CabinDeck` получен из поля `Cabin` как первая буква палубы.

Предобработка данных полностью переиспользована из 2-й лабораторной:
- категориальные признаки заполнены наиболее частым значением;
- категориальные признаки закодированы через `OrdinalEncoder`;
- числовые признаки заполнены медианой;
- выборка разделена на обучающую и тестовую в отношении 80/20 со стратификацией.

## Описание метода
Собственная реализация градиентного бустинга построена как последовательность слабых регрессионных деревьев `DecisionTreeRegressor`.

Алгоритм работает следующим образом:
1. Инициализируется константный прогноз в виде логарифма отношения шансов.
2. На каждой итерации вычисляется антиградиент логистической функции потерь.
3. По антиградиенту обучается очередное регрессионное дерево.
4. Для дерева подбирается коэффициент шага по одномерному перебору.
5. Итоговый ансамбль обновляется с учетом `learning_rate`.

Подбор гиперпараметров выполнялся через `GridSearchCV` с 5-блочной `StratifiedKFold`-кросс-валидацией по метрике `accuracy`.
