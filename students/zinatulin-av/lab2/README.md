# Лабораторная работа №2. Random Forest

## Описание метода

Random Forest — ансамблевый метод, основанный на двух идеях:

1. **Bagging (Bootstrap Aggregating)** — каждое дерево обучается на bootstrap-выборке (случайная выборка с возвращением, размером N). Объекты, не попавшие в выборку, образуют OOB-множество и используются для оценки качества без отдельного валидационного набора.

2. **Метод случайных подпространств (RSM)** — при обучении каждого дерева можно случайно выбрать подмножество признаков (`max_features='sqrt'`). Это снижает корреляцию между деревьями и повышает устойчивость ансамбля.

Итоговое предсказание — усреднение вероятностей по всем деревьям (soft voting) с выбором класса по argmax.

**OOB-оценка** позволяет оценить качество модели без выделения отдельной валидационной выборки: для каждого дерева accuracy считается на его OOB-объектах, затем берётся среднее.

**Важность признаков OOB^j** — для j-го признака выполняется случайная перестановка его значений в OOB-выборке; важность равна относительному падению OOB-accuracy.

## Датасет

Heart Disease (Statlog) из OpenML — 270 объектов, 13 признаков, бинарная классификация (наличие/отсутствие заболевания сердца). Признаки: возраст, пол, тип боли в груди, давление, холестерин, ЭКГ, пульс, наличие стенокардии и др. Разбиение: 80% обучение (216), 20% тест (54).

## Реализация

Алгоритм реализован в `source/model.py`:

- **`bagging(X, y)`** — bootstrap-выборка с возвращением; возвращает обучающие и OOB-данные.
- **`RandomForestClassifier`** — основной класс:
  - `fit(X, y)` — обучение ансамбля из `n_estimators` деревьев `DecisionTreeClassifier` (sklearn) на bootstrap-выборках.
  - `predict(X)` / `predict_proba(X)` — предсказание через усреднение вероятностей.
  - `oob_score_` — OOB accuracy (среднее по деревьям).
  - `_calculate_feature_importance()` — permutation importance через OOB^j.
  - `score()` / `get_params()` / `set_params()` — sklearn-совместимый интерфейс для `GridSearchCV`.

## Grid Search по OOB (GridSearchCV)

Сетка параметров:
```
{
    'n_estimators': [10, 50, 150],
    'max_depth': [None, 5, 10],
    'min_samples_split': [2, 5, 10],
    'min_samples_leaf': [1, 2, 4],
    'max_features': ['sqrt']
}
```

Подбор гиперпараметров выполнен через `GridSearchCV` из sklearn — метод `score()` модели возвращает OOB accuracy, что позволяет оптимизировать по OOB.

## Важность признаков (OOB^j)

Наиболее информативные признаки — тип боли в груди (chest), результат таллия-теста (thal) и число крупных сосудов (number_of_major_vessels).
