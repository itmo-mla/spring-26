# Лабораторная работа №1. Логическая классификация

## Датасет

Используется датасет Heart Attack (UCI) — 303 объекта, 13 признаков, задача бинарной классификации. Содержит категориальные (sex, cp, fbs, restecg, exang, slope, ca, thal) и количественные (age, trestbps, chol, thalach, oldpeak) признаки. Пропуски в исходных данных отсутствуют, поэтому 10% значений заменены на NaN в столбцах age, chol, thal, ca.

## Реализация

Алгоритм реализован в файле `source/model.py`. Построение дерева выполняется рекурсивной функцией `tree_growing`, которая на каждом шаге выбирает признак и порог с максимальным выигрышем (Gain) по критерию Джини.

Обработка пропусков реализована так: при обучении объекты с NaN исключаются из подсчёта Gain, при классификации — результат усредняется по обеим веткам с весами пропорциональными размерам подвыборок.

Построение и оценка дерева:

```python
tree = tree_growing(X_train, y_train, cat_features, classes)

y_pred = predict(tree, X_test, classes)
print(accuracy(y_test, y_pred))
```

Редукция дерева — обход снизу вверх, замена поддерева листом если это не ухудшает качество на валидационной выборке:

```python
prune(tree, X_val, y_val, classes)
```

Сравнение с эталонной реализацией:

```python
clf = DecisionTreeClassifier(criterion='gini')
clf.fit(X_train_filled, y_train)
```

## Результаты

| Модель | Accuracy | F1 |
|--------|----------|----|
| ID3 до редукции | 0.689 | 0.706 |
| ID3 после редукции | 0.639 | 0.667 |
| sklearn DecisionTreeClassifier | 0.721 | 0.746 |

Accuracy на обучающей выборке до редукции составляет 0.978, что указывает на переобучение.

## Анализ результатов

Редукция уменьшила дерево (с 30 до 9 листьев), но не улучшила качество на тесте. Причина — малый размер валидационной выборки (61 объект), из-за чего оценка ошибки при принятии решения об обрезке ненадёжна.

Эталонная реализация sklearn показывает несколько лучший результат. Это связано с тем, что в sklearn пропуски заполняются до обучения (медианой/модой) с применением дополнительных эвристик в реализации алгоритма. Тем не менее разница в качестве составляет несколько процентов, что для выборки из 303 объектов находится в пределах статистической погрешности.
