# Лабораторная работа №4. EM-алгоритм (GMM)

## 1. Описание алгоритма

### Gaussian Mixture Model

GMM предполагает что данные порождены смесью k гауссовых распределений. Плотность смеси:

```
p(x) = Σⱼ wⱼ · N(x; μⱼ, Σⱼ)
```

где `wⱼ` — вес компоненты, `μⱼ` — её центр, `Σⱼ` — ковариационная матрица.

### EM-алгоритм

Обучение — максимизация логарифма правдоподобия выборки:

```
L(w,θ) = Σᵢ ln Σⱼ wⱼ·N(xᵢ;μⱼ,Σⱼ) → max
```

Логарифм снаружи суммы не позволяет взять производную и получить явное решение, поэтому используется EM-алгоритм — чередование двух шагов:

**E-шаг** — вычислить ответственности `gᵢⱼ = P(j | xᵢ)` по формуле Байеса:

```
gᵢⱼ = wⱼ·N(xᵢ;μⱼ,Σⱼ) / Σₛ wₛ·N(xᵢ;μₛ,Σₛ)
```

**M-шаг** — обновить параметры с весами `gᵢⱼ`:

```
μⱼ  = Σᵢ gᵢⱼ·xᵢ / ℓ·wⱼ
Σⱼ  = Σᵢ gᵢⱼ·(xᵢ-μⱼ)(xᵢ-μⱼ)ᵀ / ℓ·wⱼ
wⱼ  = (1/ℓ) Σᵢ gᵢⱼ
```

Итерации повторяются пока `|L_t - L_{t-1}| < ε`. EM монотонно не убывает по правдоподобию на каждой итерации.

### Наивный байесовский классификатор

Байесовский классификатор выбирает класс по максимуму апостериорной вероятности:

```
a(x) = argmax P(y) · p(x|y)
```

Наивный байес — частный случай с предположением независимости признаков внутри класса: `p(x|y) = p(f₁|y)·p(f₂|y)·...·p(fₙ|y)`. Это позволяет оценивать каждый признак отдельно одномерной гауссианой, что решает проблему мультиколлинеарности и работает при малом числе объектов. 
GMM отличается тем, что не требует независимости признаков и может описывать многомодальные классы через несколько компонент.

## 3. Результаты

### Метрика качества — ПМП

Качество оценивается через среднее логарифмическое правдоподобие на тестовой выборке:

```
score = (1/ℓ) Σᵢ ln p(xᵢ)
```

Чем ближе к нулю (то есть чем больше), тем лучше модель описывает данные.

## 4. Сравнение с эталонной реализацией

При k = 2–3 разница в ПМП не превышает 0.05 нат на объект — реализация формул E-шага и M-шага корректна.

При k ≥ 4 разрыв растёт. Причина в инициализации: наша реализация делает один случайный запуск, sklearn использует инициализацию на основе k-means++, которая равномернее распределяет начальные центры. При сложном ландшафте правдоподобия это приводит к разным локальным оптимумам. Для улучшения достаточно добавить несколько перезапусков и выбрать лучший результат.

## 5. Выводы

1. При k = 2–3 собственная реализация воспроизводит результаты sklearn с разницей < 0.05 нат — алгоритм реализован корректно.
2. Основная практическая проблема GMM — чувствительность к начальной инициализации при большом k.
