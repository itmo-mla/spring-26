### SLIM (Sparse Linear Methods)

SLIM строит разреженную item-item матрицу весов W. Для каждого предмета j решается задача Elastic Net регрессии:

```
min_{w_j}  0.5 * ||r_j - R * w_j||²  +  α·l1·||w_j||₁  +  0.5·α·(1-l1)·||w_j||²

Ограничения: w_jj = 0,  w_j ≥ 0
```

Предсказание: `R_pred = R @ W`

```python
from sklearn.linear_model import ElasticNet
from sklearn.metrics.pairwise import cosine_similarity

sim = cosine_similarity(R.T)  # item-item cosine similarity

for j in range(n_items):
    r_j = R[:, j]
    sim_j = sim[j].copy()
    sim_j[j] = -np.inf
    cols = np.argsort(sim_j)[-10:]   # top-10 neighbours

    model = ElasticNet(alpha=0.00212, l1_ratio=0.5, positive=True, fit_intercept=False)
    model.fit(R[:, cols], r_j)
    W[cols, j] = model.coef_
```

### 3. Анализ и сравнение

#### Эталонная реализация

В качестве эталона использована официальная реализация [KarypisLab/SLIM](https://github.com/KarypisLab/SLIM) — реализация на C с OpenMP-параллелизацией. Параметры эталона: `l1r=1.0`, `l2r=1.0`, `niters=50`, `optTol=1e-7`.

#### Согласование параметров

`sklearn.ElasticNet` и KarypisLab/SLIM решают одну задачу, но с разной нормировкой функции потерь:

```
sklearn:      (1 / 2·n) · ||r - Rw||²  +  α·ρ·||w||₁  +  ½·α·(1-ρ)·||w||²
KarypisLab:        0.5  · ||r - Rw||²  +  l1r·||w||₁  +   ½·l2r·||w||²
```

sklearn делит квадратичный член на число пользователей `n = 943`, поэтому для эквивалентности регуляризационные коэффициенты нужно пересчитать:

```
α  = (l1r + l2r) / n  =  (1.0 + 1.0) / 943  ≈  0.00212
ρ  =  l1r / (l1r + l2r)  =  1.0 / 2.0  =  0.5
```

#### Результаты

| Модель | RMSE | NDCG@10 | Время обучения |
|---|---|---|---|
| SLIM (своя реализация) | 2.3393 | 0.9009 | 0.9 сек |
| SLIM (KarypisLab, эталон) | 2.4530 | 0.9017 | 1.0 сек |

Своя реализация с Top-10 отбором соседей показывает лучший RMSE (2.3393 против 2.4530) скорее всего за счёт того, что регрессия обучается только на наиболее релевантных предметах, а не на всех 1681.

#### NDCG@10

NDCG (Normalized Discounted Cumulative Gain) измеряет качество ранжирования:

```
DCG@K  = Σ r_i / log₂(i+1),  i = 1..K
NDCG@K = DCG@K / IDCG@K
```

Своя реализация: **0.9009**, эталон: **0.9017**.
