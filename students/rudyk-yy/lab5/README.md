### SLIM (Sparse Linear Methods)

SLIM строит разреженную item-item матрицу весов W. Для каждого предмета j решается задача Elastic Net регрессии:

```
min_{w_j}  0.5 * ||r_j - R * w_j||²  +  α·l1·||w_j||₁  +  0.5·α·(1-l1)·||w_j||²

Ограничения: w_jj = 0,  w_j ≥ 0
```

Предсказание: `R_pred = R @ W`

```python
from sklearn.linear_model import ElasticNet
from sklearn.metrics.pairwise import cosine_similarity

sim = cosine_similarity(R.T)  # item-item cosine similarity

for j in range(n_items):
    r_j = R[:, j]
    sim_j = sim[j].copy()
    sim_j[j] = -np.inf
    cols = np.argsort(sim_j)[-10:]   # top-10 neighbours

    model = ElasticNet(alpha=0.00212, l1_ratio=0.5, positive=True, fit_intercept=False)
    model.fit(R[:, cols], r_j)
    W[cols, j] = model.coef_
```

### LFM (Latent Factor Model / Funk SVD)

LFM представляет каждого пользователя вектором `p_u` и каждый фильм вектором `q_i` в пространстве T латентных факторов. Предсказание рейтинга:

```
r̂_ui = μ + b_u + b_i + p_u · q_i
```

где μ — глобальное среднее, b_u и b_i — смещения пользователя и фильма.

Обучение методом стохастического градиента (SGD):

```
ε_ui = r_ui − r̂_ui

b_u ← b_u + η (ε_ui − λ b_u)
b_i ← b_i + η (ε_ui − λ b_i)
p_u ← p_u + η (ε_ui q_i − λ p_u)
q_i ← q_i + η (ε_ui p_u − λ q_i)
```

Параметры: `n_factors=50`, `n_epochs=20`, `lr=0.005`, `reg=0.02`.

В качестве эталона использована реализация `surprise.SVD` с идентичными гиперпараметрами.

### 3. Анализ и сравнение

#### Эталонные реализации

- **SLIM**: официальная реализация [KarypisLab/SLIM](https://github.com/KarypisLab/SLIM) — C с OpenMP-параллелизацией, параметры `l1r=1.0`, `l2r=1.0`.
- **LFM**: `surprise.SVD` из библиотеки [scikit-surprise](https://surpriselib.com/).

#### Согласование параметров SLIM

`sklearn.ElasticNet` и KarypisLab/SLIM решают одну задачу, но с разной нормировкой функции потерь:

```
sklearn:      (1 / 2·n) · ||r - Rw||²  +  α·ρ·||w||₁  +  ½·α·(1-ρ)·||w||²
KarypisLab:        0.5  · ||r - Rw||²  +  l1r·||w||₁  +   ½·l2r·||w||²
```

sklearn делит квадратичный член на число пользователей `n = 943`, поэтому:

```
α  = (l1r + l2r) / n  =  (1.0 + 1.0) / 943  ≈  0.00212
ρ  =  l1r / (l1r + l2r)  =  0.5
```

#### Результаты

| Модель | RMSE | NDCG@10 | Время обучения |
|---|---|---|---|
| SLIM (своя реализация) | 2.3393 | 0.9009 | 0.8 сек |
| SLIM (KarypisLab, эталон) | 2.4530 | 0.9017 | 1.0 сек |
| LFM/Funk SVD (своя реализация) | **0.9329** | **0.9171** | 13.0 сек |
| surprise.SVD (эталон) | 0.9313 | 0.9166 | 0.3 сек |

#### NDCG@10

NDCG (Normalized Discounted Cumulative Gain) измеряет качество ранжирования:

```
DCG@K  = Σ r_i / log₂(i+1),  i = 1..K
NDCG@K = DCG@K / IDCG@K
```

## Вывод

В ходе лабораторной работы реализованы два алгоритма рекомендательных систем на датасете MovieLens 100K.

**SLIM** показал RMSE **2.3393** против **2.4530** у эталона при практически одинаковом NDCG@10 (0.9009 vs 0.9017). Высокий RMSE объясняется тем, что SLIM предсказывает почти нулевые значения для большинства пар пользователь–фильм, и после клиппинга к шкале [1,5] ошибка велика. При этом качество ранжирования (NDCG) сопоставимо с эталоном.

**LFM** существенно лучше справляется с задачей восстановления рейтингов: RMSE **0.9329** против **0.9313** у `surprise.SVD` — расхождение менее 0.002, что говорит о корректности реализации. NDCG@10 нашей реализации (0.9171) даже незначительно превышает эталон (0.9166). Время обучения LFM (13 сек) значительно выше, чем у оптимизированного `surprise.SVD` (0.3 сек), что ожидаемо для чистой Python-реализации SGD без векторизации внутреннего цикла.
